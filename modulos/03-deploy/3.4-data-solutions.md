## Cloud Storage — classes e quando usar

```
Structured data? → Use serviço de DB (Spanner, Cloud SQL, etc.)
Unstructured data?
  ├─ Acesso < 1x/ano?       → Archive
  ├─ Acesso < 1x/90 dias?   → Coldline
  ├─ Acesso < 1x/30 dias?   → Nearline
  └─ Acesso frequente?      → Standard
```

| Classe | Duração mínima | Cobrança de retrieval | Quando usar |
|--------|----------------|----------------------|-------------|
| **Standard** | Nenhuma | ❌ | Dados ativos, acesso frequente |
| **Nearline** | **30 dias** | ✅ | Backup mensal |
| **Coldline** | **90 dias** | ✅ | Backup trimestral |
| **Archive** | **365 dias** | ✅ | Compliance, arquivamento legal |

## Cloud Storage — Location types

| Tipo | O que é | Uso |
|------|---------|-----|
| **Regional** | Uma região | Minimiza latência local |
| **Dual-region** | Par de regiões no **mesmo continente** | Geo-redundância |
| **Multi-region** | Área ampla (US, EU, ASIA) | Disponibilidade máxima |

---

## Cloud Storage — criar bucket

```bash
# Criar bucket (padrão atual)
gcloud storage buckets create gs://BUCKET_NAME [--location=LOCATION]

# Sem --location → default: US multi-region
```

**Pegadinhas:**
- `gsutil` está sendo **descontinuado** — prefira `gcloud storage`
- `--placement` (dual-region) só aceita regiões do **mesmo continente**
- Bucket sem `--location` → **US multi-region** por default

## Cloud SQL — características

| Aspecto | Detalhe |
|---------|---------|
| **Tipo** | Managed relational DB |
| **Suporta** | MySQL, PostgreSQL, SQL Server |
| **Gerencia** | Backups, HA, encryption, updates, monitoring |
| **Limitação crítica** | **NÃO suporta UDFs** (User-Defined Functions) |
| **Region/version** | **Não pode mudar após criar** |

---

## Cloud SQL — setup de HA (cai na prova!)

| Flag | Função |
|------|--------|
| **`--availability-type=regional`** ✅ | **Failover automático** para zona standby |
| `--availability-type=zonal` | Sem failover (default) |
| `--replica-type` | Tipo de read replica (não automatiza failover) |
| `--master-instance-name` | Cria **read replica** (replica dados, NÃO faz failover) |

```bash
gcloud sql instances create NAME \
  --availability-type=regional \
  --database-version=MYSQL_8_0 \
  --region=us-central1
```

## BigQuery — métodos de ingestão

| Método | Custo | Quando usar |
|--------|-------|-------------|
| **Batch load job** | **Grátis** | CSV, JSON, Avro, Parquet, ORC — one-time |
| **BigQuery Data Transfer Service** | **Grátis** | **Slowly changing data, agendado** ✅ |
| **bq load + cron** | Grátis | Grátis, mas mais complexo que DTS |
| **Streaming API** | **Pago por volume** | Real-time, queryable imediato |
| **Dataflow + Apache Beam** | Pago (recursos) | ETL complexo |
| **Cloud Composer** (Airflow) | Pago | Pipelines complexas com orquestração |

**Formatos aceitos em batch:** CSV, JSON, Avro, Parquet, ORC

## DQ 06 — Bucket para NY + SF (só EUA), sem ACLs

**Resposta: C — `gcloud storage buckets create` SEM `--location`**

- Sem `--location` → **US multi-region** por default → cobre NY + SF ✅
- Não usa ACLs por default (Uniform Bucket-Level Access)

## DQ 07 — Cloud SQL com failover automático em outage zonal

**Resposta: A — `--availability-type=regional`**

- Único flag que configura **failover automático** para zona standby
- `--replica-type` e `--master-instance-name` criam réplicas mas não automatizam failover

## DQ 08 — Slowly changing data → BigQuery, menor custo, menos passos

**Resposta: D — BigQuery Data Transfer Service**

| Opção | Por que não? |
|-------|--------------|
| `bq load + cron` | Grátis mas **mais passos** (script + agendador) |
| Streaming API | ❌ **Pago** por volume |
| Dataflow | ❌ **Pago** por recursos |
| **Data Transfer Service** ✅ | Grátis + **um comando** + agendado nativamente |

## Pegadinhas de prova

| Pegadinha | Resposta |
|-----------|----------|
| "MySQL com UDFs" | **Compute Engine + MySQL** (Cloud SQL não suporta UDFs) |
| "Failover automático Cloud SQL" | `--availability-type=regional` |
| "Read replica Cloud SQL" | `--master-instance-name` (não faz failover) |
| "Bucket padrão cobre NY + SF" | Sem `--location` → US multi-region |
| "Slowly changing data → BigQuery" | **Data Transfer Service** (grátis + simples) |
| "`gsutil` ou `gcloud storage`?" | **`gcloud storage`** (gsutil sendo descontinuado) |
| "Dual-region entre continentes" | ❌ Não existe — só **mesmo continente** |
