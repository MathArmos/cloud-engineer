# 3.6 — Implementing Resources via Infrastructure as Code

> **DQ:** 10 · **Curso:** Build Infrastructure with Terraform on Google Cloud _(opcional)_

---

## Por que IaC?

- **Declarativo**: você diz o *quê*, a tool decide o *como*
- **Repetível e versionável** — infraestrutura no git
- Reset de ambientes dev/test
- DevOps best practices

---

## Ferramentas IaC no GCP

| Ferramenta | Para que serve |
|-----------|----------------|
| **Terraform** | Open source, declarativo, multi-cloud |
| **Cloud Foundation Toolkit** | Templates Google-best-practice para Terraform/Helm |
| **Config Connector** | Gerencia GCP via objetos Kubernetes |
| **Helm** | Package manager para Kubernetes |

---

## Terraform no GCP — arquivos principais

| Arquivo | Função |
|---------|--------|
| `main.tf` | Configuração principal dos recursos |
| `backend.tf` | Remote state (aponta para GCS bucket com versioning) |
| `terraform.tfstate` | State local |
| `cloudbuild.yaml` | Instruções para Cloud Build automatizar Terraform |

**Config files em GCP vivem em:** Cloud Storage bucket com **versioning enabled**  
**Execução automatizada via:** Cloud Build (submete comandos Terraform via YAML)

---

## Lifecycle de comandos

```
terraform init  →  terraform plan  →  terraform apply
                                            ↓
                                  terraform destroy (quando descartar)
```

| Comando | O que faz |
|---------|-----------|
| **`terraform init`** | Baixa a última versão do **provider** |
| **`terraform plan`** | Verifica sintaxe + preview do que será criado/alterado |
| **`terraform apply`** | **Cria/configura recursos** declarados no config file |
| **`terraform destroy`** | Destrói todos os recursos do config file |

---

## DQ 10 — O que o `terraform apply` faz?

**Resposta: D — Sets up resources requested in the terraform config file**

| Comando | Confusão comum |
|---------|----------------|
| `init` | Baixa **provider** (não cria recursos) |
| `plan` | **Preview** (não cria recursos) |
| **`apply`** ✅ | **Cria/configura recursos** |
| `destroy` | Remove recursos |

## Pegadinhas de prova

| Pegadinha | Resposta |
|-----------|----------|
| "O que `terraform apply` faz?" | **Cria recursos** (não é `init`) |
| "O que `terraform init` faz?" | **Baixa o provider** (não cria recursos) |
| "O que `terraform plan` faz?" | **Preview** — não toca infraestrutura |
| "State remoto no GCP" | Cloud Storage bucket com **versioning enabled** |
