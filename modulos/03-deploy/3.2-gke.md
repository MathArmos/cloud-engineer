# 3.2 — Deploying and Implementing GKE Resources

> **DQ:** 03 · **Curso:** Getting Started with Google Kubernetes Engine

---

## Conceitos fundamentais

- **Nodes**: VMs Compute Engine que rodam containers
- **Cluster**: conjunto de nodes + control plane gerenciado pelo Google
- **API K8s**: você descreve o estado desejado, o K8s faz acontecer

---

## Autopilot vs Standard

| Característica | **Autopilot** | **Standard** |
|----------------|---------------|--------------|
| Provisionamento | Fully-managed pelo Google | Você define e gerencia |
| Cobrança | Por **recursos dos pods** | Por **nodes** |
| Localização | **Regional sempre** | **Zonal ou regional** |
| Gerenciamento | Nível de pod | Nível de cluster |
| Image type | Apenas **Container-Optimized OS** | Ubuntu, COS, etc. |
| Custom packages | ❌ | ✅ |
| Flexibilidade de arquitetura | ❌ Limitada | ✅ Total |

**Regra mnemônica:** Autopilot = Auto**pilot** = você não pilota nada = só regional, só COS, sem customização.

---

## Availability — Zonal vs Regional

| Tipo | Control Plane | Nodes |
|------|---------------|-------|
| **Zonal** | Único, em uma zona | Pode distribuir em múltiplas zonas |
| **Regional** | Réplicas em múltiplas zonas | Replicados em **3 zonas** (configurável) |

---

## Network Routing

| Tipo | Como funciona | Quando usar |
|------|---------------|-------------|
| **VPC-native** | Pods usam **alias IPs** | Obrigatório para Internal HTTP(S) LB |
| **Routes-based** | Usa Google Cloud Routes | Legacy |

---

## Network Isolation

| Tipo | Característica |
|------|----------------|
| **Public GKE** | Nodes e pods acessíveis por IPs públicos |
| **Private GKE** | Pods/nodes usam IPs internos — isolados de redes públicas |

---

## Versões e upgrades

- Você escolhe versão específica **ou** release channel: **Rapid, Regular, Stable**
- Sem escolha → versão default atual
- **Best practice**: habilitar **auto-upgrade** de nodes e cluster

---

## Comandos essenciais

```bash
# Criar cluster Standard (zonal)
gcloud container clusters create CLUSTER \
  --zone=us-central1-a \
  --image-type=UBUNTU

# Criar cluster Autopilot (regional — único modo possível)
gcloud container clusters create-auto CLUSTER \
  --region=us-central1

# Apontar kubectl para o cluster
gcloud container clusters get-credentials CLUSTER --zone=us-central1-a

# Deploy de app
kubectl apply -f manifest.yaml
```

## DQ 03 — Cluster piloto, privado, não-HA, flexível para mudar arquitetura

**Resposta: B — Private Standard zonal em us-central1-a + default pool + Ubuntu**

| Opção | Por que está errada |
|-------|---------------------|
| Autopilot us-central1-a | ❌ Autopilot **é sempre regional** — us-central1-a é uma zona |
| Container-Optimized OS | ❌ Não suporta custom packages |
| Autopilot + Ubuntu | ❌ Autopilot não suporta Ubuntu |
| **Standard zonal + Ubuntu** ✅ | Zonal = não-HA; Standard = flexível; Ubuntu = custom packages |

---

## Pegadinhas de prova

| Pegadinha | Resposta |
|-----------|----------|
| "Cluster em us-central1-a com Autopilot" | ❌ Impossível — Autopilot é sempre **regional** |
| "Cluster com Ubuntu" | Só **Standard** — Autopilot não suporta Ubuntu |
| "Piloto, pequeno, não-HA" | **Standard zonal** (não Autopilot, não regional) |
| "Controle total da arquitetura" | **Standard** — Autopilot é limitado |
