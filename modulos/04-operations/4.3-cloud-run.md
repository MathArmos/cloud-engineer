# 4.3 — Managing Cloud Run Resources

> **DQ:** 06 · **Curso:** Developing Applications with Cloud Run: Fundamentals

---

## Autoscaling — settings e defaults

| Setting | Default | Função |
|---------|---------|--------|
| **CPU utilization** | **60%** | Threshold para scale up (não limita conexões) |
| **Concurrency** | **80** (max 1000) | Requests simultâneas por instância |
| **Max instances** | **1000** | Teto de instâncias — limita custo e conexões a backends |
| **Min instances** | **0** | Piso — mantém instâncias quentes, evita cold start |

**Idle behavior:** instâncias podem ficar idle até **15 minutos** após startup sem cobrança. Por default, escala para **zero** quando sem tráfego.

---

## Como limitar conexões a um banco backend?

**Resposta: definir `Max Instances`**

| Setting | Por que não resolve o problema? |
|---------|--------------------------------|
| Min instances | É o **piso** — garante mínimo, não limita máximo |
| CPU utilization | Não afeta conexões a backing services |
| Concurrency | Limita requests **por instância**, não total de conexões |
| **Max instances** ✅ | Limita total de instâncias → limita pool agregado de conexões |

**Raciocínio:** cada instância de Cloud Run abre suas próprias conexões ao DB. Limitar o número de instâncias = limitar conexões totais ao banco.

## Traffic splitting — canary e rollback

```bash
# Dividir tráfego entre revisions (canary: 20% nova, 80% atual)
gcloud run services update-traffic SERVICE \
  --to-revisions=REVISION=20,LATEST=80 \
  --region=REGION

# Rollback: 100% para revision anterior
gcloud run services update-traffic SERVICE \
  --to-revisions=OLD_REVISION=100 \
  --region=REGION
```

## Cloud Run vs Cloud Run Functions

| | **Cloud Run** | **Cloud Run Functions** |
|--|---------------|------------------------|
| Conexões por instância | **Múltiplas concorrentes** | **Uma por instância** |
| Tipo de carga | Containers completos | Código funcional |
| Cobrança | Por requests/recursos | Por requests |

---

## Comandos essenciais

```bash
# Deploy nova versão
gcloud run deploy SERVICE \
  --image=IMAGE \
  --region=REGION

# Configurar autoscaling
gcloud run services update SERVICE \
  --max-instances=10 \
  --min-instances=1 \
  --concurrency=50 \
  --region=REGION

# Ver revisões
gcloud run revisions list --service=SERVICE --region=REGION
```

## DQ 06 — Limitar conexões ao banco de dados

**Resposta: Max Instances**

O banco tem limite de conexões simultâneas. Cada instância do Cloud Run cria conexões independentes. Logo, para limitar conexões ao banco: **limitar Max Instances**.

---

## Pegadinhas de prova

| Pegadinha | Resposta |
|-----------|----------|
| "Limitar conexões ao DB" | **Max instances** (não concurrency, não CPU utilization) |
| "Evitar cold start" | **Min instances > 0** (paga mesmo idle) |
| "Default de concurrency" | **80** requests por instância |
| "Default de CPU utilization" | **60%** threshold de scale up |
| "Escala para zero por padrão?" | ✅ Sim (min instances default = 0) |
