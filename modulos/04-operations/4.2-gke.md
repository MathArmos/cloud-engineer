## Objetos Kubernetes — hierarquia

```
Deployments
   ↓ gerencia e monitora
  Pods (réplicas)
   ↑ expostos via
Services
```

| Objeto | Função | Detalhe crítico |
|--------|--------|-----------------|
| **Pod** | Menor unidade implantável | 1+ containers, IP interno **mutável** |
| **Deployment** | Gerencia set de pods idênticos | Usa ReplicaSet, faz **rolling upgrade** ao atualizar template |
| **Pod template** | Spec *dentro* de um Deployment | Não é objeto independente |
| **Service** | Expõe pods com IP estável | Seleciona pods por **labels** |

## Load Balancers no GKE

| O que criar | Objeto K8s | Annotation | Camada | Escopo |
|-------------|-----------|-----------|--------|--------|
| **Network LB externo** | `Service: type: LoadBalancer` | — | **L4** | Regional |
| **External HTTP(S) LB** | `Ingress` | `ingress.class: "gce"` | **L7** | Global |
| **Internal HTTP(S) LB** | `Ingress` | `ingress.class: "gce-internal"` + **NEG** | **L7** | Regional |

### Internal HTTP(S) LB — detalhes (cai muito!)

- Annotation obrigatória no **Service**:
  ```yaml
  cloud.google.com/neg: '{"ingress": true}'
  ```
- Proxies ficam em **proxy-only subnet** na região do VPC
- Cluster precisa ser **VPC-native** (alias IPs nos pods) — não funciona em routes-based

### External Traffic Policy no Service `type: LoadBalancer`

| Policy | Comportamento |
|--------|---------------|
| **Cluster** | Tráfego para qualquer nó saudável; kube-proxy roteia internamente |
| **Local** | Só nodes com o pod recebem tráfego; **preserva source IP** |

## `kubectl` — imperativo vs declarativo

| Estilo | Comandos | Quando usar |
|--------|----------|-------------|
| **Declarativo** | `apply` | Idempotente, trabalha em diretório, estado desejado — **preferido** |
| **Imperativo** | `run`, `create`, `replace`, `delete` | One-off, opera em objeto único |

### Diferenças finas entre comandos imperativos

| Comando | O que faz |
|---------|-----------|
| `kubectl run` | Cria objeto via **argumentos CLI** |
| `kubectl create` | Cria via **manifest** — falha se já existe |
| `kubectl replace` | Substitui objeto completo |
| `kubectl apply` | Cria **OU** atualiza — idempotente ✅ |
| `kubectl get` | Lista recursos |
| `kubectl expose` | Cria Service que roteia para pods com labels |

## Estrutura de manifests importantes

**Deployment:**
```yaml
kind: Deployment
spec:
  replicas: 3
  template:
    metadata:
      labels:
        app: myapp
    spec:
      containers:
        - name: myapp
          image: gcr.io/project/image:tag
```

**Service:**
```yaml
kind: Service
spec:
  selector:
    app: myapp      # precisa casar TODOS os labels
  ports:
    - port: 80        # porta do Service
      targetPort: 8080 # porta do container
```

## Comandos essenciais

```bash
# Listar recursos
kubectl get pods
kubectl get services
kubectl get deployments

# Deploy declarativo (preferido)
kubectl apply -f manifest.yaml

# Ver logs de um pod
kubectl logs POD_NAME

# Exec dentro de um container
kubectl exec -it POD_NAME -- bash

# Escalar deployment
kubectl scale deployment DEPLOY --replicas=5

# Atualizar image (rolling update)
kubectl set image deployment/DEPLOY container=NEW_IMAGE
```

## Pegadinhas de prova

| Pegadinha | Resposta |
|-----------|----------|
| "Internal HTTP(S) LB no GKE" | `gce-internal` annotation + **NEG** annotation no Service + cluster **VPC-native** |
| "Imperativo vs declarativo" | `apply` é declarativo; `create`/`replace` são imperativos |
| "`kubectl create` com objeto existente" | ❌ Falha — use `apply` |
| "Source IP preservado no LB" | External Traffic Policy = **Local** |
| "IP do pod é estável?" | ❌ Pod IP é mutável — use **Service** para IP estável |
| "Service seleciona pods por..." | **Labels** (todos precisam casar) |
