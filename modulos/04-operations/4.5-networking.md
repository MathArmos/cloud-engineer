# 4.5 — Managing Networking Resources

> **DQ:** 08 · **Curso:** Essential GCP Infrastructure: Foundation

---

## Expandir subnet — regras

| Característica | Detalhe |
|----------------|---------|
| **Direção** | Só **expande** (prefix menor = mais IPs). Nunca encolhe. |
| **Downtime** | **Zero** — sem interrupção |
| **Sintaxe** | Reduz o prefix-length (ex: /24 → /20) |

```bash
gcloud compute networks subnets expand-ip-range SUBNET \
  --region=REGION \
  --prefix-length=PREFIX_LENGTH
```

---

## Cálculo de IPs — fórmula

> **IPs usáveis = 2^(32 − prefix) − 4**

GCP reserva **4 IPs** em qualquer subnet:
- Network address (primeiro)
- Default gateway (segundo)
- Penúltimo (reservado pelo Google)
- Broadcast (último)

| Prefix | Total IPs | Usáveis |
|--------|-----------|---------|
| /24 | 256 | 252 |
| /23 | 512 | 508 |
| /22 | 1024 | 1020 |
| /21 | 2048 | **2044** ← cabe 2000 VMs |
| /20 | 4096 | 4092 |

**Regra rápida:** a cada bit que você remove do prefix, os IPs **dobram**.

---

## IPs ephemeral vs static

| Tipo | Característica |
|------|----------------|
| **Ephemeral** | Default para IPs internos e externos — muda quando recurso é recriado |
| **Static** | Persiste — reserve quando precisar de IP fixo (DNS, SSL, etc.) |

```bash
# Reservar IP estático externo
gcloud compute addresses create NAME --region=REGION

# Reservar IP estático interno
gcloud compute addresses create NAME \
  --region=REGION \
  --subnet=SUBNET \
  --addresses=10.0.0.100
```

## Cloud DNS — gerenciar registros

```bash
# Criar zona gerenciada
gcloud dns managed-zones create ZONE_NAME \
  --description="descrição" \
  --dns-name=example.com.

# Adicionar registro A
gcloud dns record-sets create www.example.com. \
  --zone=ZONE_NAME \
  --type=A \
  --ttl=300 \
  --rrdatas=IP
```

## Cloud NAT — pontos críticos

- Permite VMs **sem IP externo** acessarem a internet (outbound only)
- **Não recebe conexões de entrada** — só saída
- Configurado por região + Cloud Router
- VMs usam NAT IP compartilhado automaticamente

## DQ 08 — Expandir subnet para acomodar 2000 VMs

**Raciocínio:**
- Atual: /24 → 252 usáveis (insuficiente)
- Necessário: pelo menos 2000 usáveis
- /21 → 2044 usáveis ✅

**Resposta:** `gcloud compute networks subnets expand-ip-range SUBNET --prefix-length=21`

## Pegadinhas de prova

| Pegadinha | Resposta |
|-----------|----------|
| "Encolher subnet" | ❌ Impossível — só expande |
| "Expandir subnet causa downtime?" | ❌ Zero downtime |
| "Prefix menor = mais ou menos IPs?" | **Mais IPs** (prefix menor = menos bits fixos) |
| "/21 cabe 2000 VMs?" | ✅ 2044 usáveis |
| "VM sem IP externo precisa de internet" | **Cloud NAT** |
| "Cloud NAT aceita conexões de entrada?" | ❌ Outbound only |
