> **DQs:** 01, 02 · **Curso:** Essential GCP Infrastructure: Foundation

## Snapshots — o que são

| Característica | Detalhe |
|----------------|---------|
| **Natureza** | **Incremental** (mais barato que imagens completas) |
| **Aplica-se a** | Apenas **persistent disks** (não local SSD) |
| **Armazenamento** | Cloud Storage gerenciado pelo snapshot service |
| **Compressão** | Automática |
| **Localização** | Regional ou multi-regional (afeta custo) |
| **Live snapshots** | ✅ Pode criar com VM rodando |
| **Frequência mínima** | **≥ 10 minutos** entre snapshots do mesmo disco |
| **Restore** | Pode restaurar em **zona/região diferente** → usado para mover VMs |

---

## Como snapshots incrementais funcionam

- **Snapshot 1** = full backup (tudo)
- **Snapshot 2** = só blocos modificados desde Snapshot 1
- **Snapshot 3** = só blocos modificados desde Snapshot 2
- Às vezes o sistema cria um novo full para limpar dependências (otimização interna)

---

## Deletar snapshots — cuidado!

- Ao deletar snapshot N, seus dados únicos são **copiados para os snapshots downstream** que dependem dele → eles **aumentam de tamanho** (mas nenhum dado é perdido)
- **Não é possível deletar snapshot schedule** enquanto ainda está **attached a um PD** → faça **detach primeiro**

---

## Padrão de comandos `gcloud`

> `gcloud <grupo> <subgrupo> <verbo>`  
> O verbo (**list, describe, delete**) sempre vem **no final**.  
> Para snapshots: verbos disponíveis são **list, describe, delete** — não existe `get`.

```bash
# Listar snapshots do projeto
gcloud compute snapshots list --project PROJECT_ID

# Descrever snapshot (creation time, size, source disk)
gcloud compute snapshots describe SNAPSHOT_NAME

# Criar snapshot de um disco
gcloud compute disks snapshot DISK_NAME \
  --zone=ZONE \
  --snapshot-names=SNAPSHOT_NAME

# Deletar snapshot
gcloud compute snapshots delete SNAPSHOT_NAME
```

---

## Imagens vs Snapshots

| | **Snapshot** | **Imagem** |
|--|-------------|-----------|
| **Uso** | Backup incremental de disco | Template para criar novas VMs |
| **Escopo** | Específico de um disco | Pode ser compartilhada entre projetos |
| **Custo** | Mais barato (incremental) | Mais caro (completa) |
| **Restaura em** | Qualquer zona/região | Qualquer zona/região |

## Pegadinhas de prova

| Pegadinha | Resposta |
|-----------|----------|
| "Deletar snapshot com schedule" | **Detach o schedule primeiro**, depois delete |
| "Snapshots de local SSD" | ❌ Impossível — só persistent disks |
| "Mover VM para outra região" | Crie snapshot → restore em nova zona/região |
| "Verbo gcloud para snapshot" | `list`, `describe`, `delete` (não `get`) |
| "Intervalo mínimo entre snapshots" | **10 minutos** |
| "Deletar snapshot reduz tamanho total?" | ❌ Não — dados são copiados para downstream |
